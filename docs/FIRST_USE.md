# 首次使用指南

这份指南面向第一次 clone、部署或打开 XXSG 的用户。

## 1. 启动项目

```bash
npm install
npm run serve
```

访问：

```text
http://localhost:8080
```

如果部署在 Vercel、Netlify、GitHub Pages 等平台，直接打开部署后的站点地址即可。

## 2. 创建本地账号

开源版本无需联系作者获取账号。首次使用时：

1. 打开 `login.html`。
2. 点击“本地注册 / 创建账号”，进入 `register.html`。
3. 填写用户名、可选邮箱和密码。
4. 提交后会创建当前浏览器可用的本地账号，并自动进入应用。

这个账号默认保存在浏览器本地，不会上传到作者、GitHub 或其它云端服务。

## 3. 可选：初始化本地管理员

如果你需要进入管理员后台配置 AI Key 或管理本地用户：

1. 打开 `login.html`。
2. 进入管理员登录。
3. 用户名填写 `admin`。
4. 输入至少 8 位密码。
5. 第一次成功登录会把该密码保存为当前浏览器里的本地管理员密码。

请妥善保存这个密码。它属于当前浏览器本地数据的一部分，清理浏览器数据后可能需要重新初始化。

如果页面提示“当前浏览器已经初始化过本地管理员”，说明这个浏览器里已经保存过管理员密码。请使用之前设置的密码登录；如果忘记了，可以点击登录页里的“重置本地管理员”，清除当前浏览器的管理员密码后重新初始化。

进入管理员后台后，也可以创建普通用户并分配功能权限。

注意：当前开源版本没有服务端账号系统。用户数据默认存储在浏览器本地，不会自动同步到其它设备。

## 4. 使用每日一签

每日一签支持两种模式：

- 传统签文：无需 API Key。
- AI 签文：需要进入管理员后台配置 DeepSeek API Key。

如果没有配置 AI Key，传统签文仍可正常使用。

## 5. 配置 AI Key（BYOK）

当前开源版采用 BYOK（Bring Your Own Key）模式。

你需要：

1. 自行在模型服务商处申请 API Key。
2. 登录管理员后台配置。
3. 确认已启用对应 AI 功能。

配置入口：

1. 打开 `login.html`。
2. 切换到“管理员后台登录”。
3. 使用管理员账号 `admin` 登录；如果是首次使用，输入任意 8 位以上密码完成本地管理员初始化。
4. 进入管理员后台后：
   - 每日一签 AI：打开“AI 签语配置”，填写 DeepSeek API Key。
   - AI 任务分析 / 智能建议：打开“AI 服务配置”，选择 DeepSeek 或 OpenAI，填写对应 API Key 并启用。

当前支持的 API Key：

| 功能 | 支持的 API Key | 默认接口 / 模型 |
| --- | --- | --- |
| 每日一签 AI 签文 | DeepSeek API Key | `https://api.deepseek.com/chat/completions`，`deepseek-chat` |
| AI 任务分析 / 智能建议 | DeepSeek API Key | `https://api.deepseek.com/chat/completions`，`deepseek-chat` |
| AI 任务分析 / 智能建议 | OpenAI API Key | `https://api.openai.com/v1/chat/completions`，`gpt-3.5-turbo` |

说明：当前静态开源版主要配置 API Key，默认使用项目内置的官方接口和模型名称；它不是完整的自定义模型网关。

请不要把真实 API Key 写入源码或提交到 GitHub。

## 6. 备份数据

由于当前版本是本地优先应用，请在以下操作前导出备份：

- 清理浏览器缓存；
- 更换浏览器；
- 更换设备；
- 重装系统；
- 重置站点数据。

## 7. 常见问题

### 这是完整 SaaS 吗？

不是。当前版本适合个人、自托管和本地优先使用。如果你需要公网多人账号、跨设备同步、服务端管理员权限和统一 AI 额度，请参考 `docs/saas-backend-roadmap.md`。

### 我能直接部署给别人用吗？

可以作为静态应用给别人体验，但请明确告知：数据存在用户自己的浏览器里，AI Key 需要用户或部署者自己配置。

### 我能用自己的 API Key 给所有公网用户使用吗？

不建议。当前前端 BYOK 模式无法真正隐藏供应商 API Key。若要统一提供 AI 能力，请先实现服务端 AI Proxy。
