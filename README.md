# XXSG（象限时光）

XXSG 是一个本地优先的个人生产力 PWA，围绕四象限任务管理，提供任务、日历、习惯打卡、番茄钟、时间统计、每日一签和 AI 辅助能力。

> 当前开源版本定位为“本地优先 / 自托管 / BYOK（Bring Your Own Key）”。
> 它不是完整的公网多人 SaaS：用户、任务、设置和 AI 配置默认保存在浏览器本地存储中。

## 功能概览

- 四象限任务管理：任务分类、子任务、置顶、拖拽排序、列表/象限视图切换。
- 日历日程：日历事件管理，并支持从任务同步到日历。
- 习惯打卡：日历式打卡、连续天数和趋势统计。
- 番茄钟：自定义专注/休息时长、音效和历史记录。
- 数据看板：任务数量、完成率、象限分布等统计。
- 每日一签：传统签文与可选 AI 签文。
- AI 辅助：任务分析、智能建议等能力，需要使用者自行配置 API Key。
- 数据导入/导出：用于本地备份和迁移。

## 重要安全说明

### 1. 本项目默认不提供云端账号系统

当前版本主要依赖浏览器本地存储。换浏览器、清理缓存或更换设备前，请先导出备份。

如果你需要公网多人注册、统一登录、跨设备同步、管理员服务端鉴权，请参考 `docs/saas-backend-roadmap.md`，先接入后端认证和数据库。

### 2. AI 功能采用 BYOK 模式

每日一签 AI 和 AI 服务需要配置第三方模型 API Key。当前本地优先版本中，API Key 会保存在部署者/使用者自己的浏览器侧存储中。

请不要把你自己的 API Key 写进源码，也不要把配置后的浏览器数据导出给他人。

如果你要给公网用户统一提供 AI 能力，请先实现服务端 AI Proxy，把模型供应商 Key 放在服务器环境变量里，并增加鉴权、限流和额度控制。

### 3. 管理员密码由部署者初始化

开源版本不再内置公开默认管理员密码。首次自托管使用时，请使用管理员账号 `admin` 登录，并输入至少 8 位密码完成本地初始化。

这个管理员机制仍然属于本地前端管理，适合个人/自托管使用，不等同于服务端安全管理员系统。

## 快速开始

```bash
npm install
npm run serve
```

然后访问：

```text
http://localhost:8080
```

也可以直接用任意静态站点服务部署，例如 GitHub Pages、Vercel、Netlify、Cloudflare Pages 等。

## 部署建议

### 静态部署

适合个人使用、自托管演示和开源体验。

- 不要提交任何真实 API Key。
- 不要提交浏览器导出的个人数据备份。
- AI Key 由每个部署者/使用者自行配置。
- 管理员密码由首次使用者本地初始化。

### Vercel API 示例

仓库包含简单的 `api/telemetry.js` 和 `api/stats.js` 示例。`/api/stats` 需要配置环境变量：

```text
STATS_ADMIN_KEY=replace-with-a-random-secret
```

未配置该变量时，统计查看接口默认不可用。
