# 安全策略

## 支持范围

当前开源版本是本地优先 PWA，安全边界与完整 SaaS 不同。

我们重点关注：

- 真实密钥不应进入仓库；
- 默认管理员密码不应公开内置；
- 用户会话和备份不应保存明文敏感字段；
- 页面渲染应避免不必要的 HTML 字符串注入；
- AI Key 的 BYOK 边界应被清晰说明。

## 报告安全问题

如果你发现安全问题，请优先通过 GitHub 私密渠道联系维护者，或创建一个不包含可利用细节的 issue。

请在报告中包含：

- 影响范围；
- 复现步骤；
- 预期结果与实际结果；
- 你建议的修复方向。

## 重要边界

当前版本不承诺：

- 服务端账号安全；
- 多租户数据隔离；
- 服务端管理员鉴权；
- 统一隐藏第三方 AI Provider Key；
- 公网 SaaS 级别的审计和限流。

如需这些能力，请参考 `docs/saas-backend-roadmap.md`。
